webpackのloader周りの話です

自作したい。バグ or よく分からん現象踏んだという際に参考ください

loaderの処理順

use: [ 'abc-loader', 'def-loader' ]

def-loader => abc-loader

正確な処理順

abc-loader.pitch => def-loader.pitch => def-loader default => abc-loader default

pitchがString or Bufferを返却すると対象のdefaultの処理まで省略される

abc-loader.pitch return "abc" (省略) def-loader.pitch (省略) def-loader.loader (省略) abc-loader.pitch

例2

abc-loader.pitch return def-loader.pitch return "abc" (省略) def-loader.loader abc-loader.pitch return "something"

どんなことに使われている?

cache-loader

pitchフェイズでcacheファイルのmtimeと依存ファイルのmtimeを比較、差異がない場合はcacheファイルを読みこみreturn

defaultフェイズで依存ファイルのpathとそのmtime、前のloaderで生成した文字列を保存

mtime … 最終変更時刻 (modify time)

最後にファイルを変更した時刻のこと。正確にはファイルに write(2) または truncate(2) した場合に変更
